# ubiquitous-language — ユビキタス言語（用語集）

仕様（discover/specify）・設計（plan）・実装（implement）・テストで**同じ言葉を同じ意味で使う**ための用語台帳。これは生きた辞書で、機能を発見するたびに育てる。（**運用方法は汎用**。下の用語表そのものは本プロジェクト固有のデータで、他プロジェクトでは差し替える。）

## 使い方

- どのタスクでも**最初に参照**して用語を揃える。
- discover/specify で**新語が出たら、ここへ追加提案**する（定義＋英語識別子）。表記ゆれを作らない。
- 文書は日本語＋英語併記、**コードの識別子は英語**（下表の英語名に従う）。

## 用語（初期セット）

| 日本語 | 英語（識別子） | 定義 |
| --- | --- | --- |
| 的 | target | クリック対象。画面上の位置とサイズ（半径）を持つ。 |
| 的の出現 | spawn | 的が画面に現れること。出現時刻が反応時間の起点。 |
| ヒット | hit | 的の内側でのクリック。 |
| ミス | miss | 的の外側でのクリック。 |
| 反応時間 | reaction_time | 的の出現からクリックまでの時間（ミリ秒, ms）。 |
| 命中率 | accuracy | ヒット数 ÷ total_clicks（総クリック数）。 |
| 総クリック数 | total_clicks | accuracy の分母。hit ＋ 空白 miss の合計。hit 済み的の再クリック・キャンバス外/HUD・セッション終了後のクリックは数えない。 |
| セッション | session | 1回の練習単位（開始〜終了）。制限時間または的数で区切る。 |
| スコア | score | セッションの成績記録（命中率・平均反応時間・日時など）。 |
| 銃 | gun | 挙動パラメータ（連射速度・反動・的サイズ）を持つマスタ。 |
| 連射速度 | fire_rate | 単位時間あたりに発射できる数。 |
| 反動 | recoil | 発射後に照準がずれる挙動。 |
| 照準 | aim_point | 現在の狙点（クリック位置の基準）。 |

## 規約

- 1概念=1用語。同義語を増やさない（「正解」「当たり」ではなく **hit** に統一、等）。
- **命名規則（言語ごとに大小文字を変える。語幹は共通）**: Python／DB＝`snake_case`（`reaction_time`）、TypeScript＝`camelCase`（`reactionTime`）、型・クラス＝`PascalCase`（`Session`）。同じ概念は同じ語幹で表す。
- これらは plan 段階で**ドメインのエンティティ／値オブジェクト**にほぼ対応する（例: `Session`・`Score`・`Gun` はエンティティ、`ReactionTime`・`Accuracy` は値オブジェクト候補）。
- DB テーブル名・カラム名・API のフィールド名も、原則この英語名（言語に応じた大小文字）に揃える。

## メンテナンス

新機能の discover で概念が増えたら、この表に行を足してから先へ進む（用語未定義のまま仕様を書かない）。
